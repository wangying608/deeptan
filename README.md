## About

DeepTAN is a graph-based multi-task framework designed to infer large-scale multi-omics trait-associated networks (TANs) and reconstruct phenotype-specific omics states.

## Installation

```bash
conda create -n deeptan python=3.13 -y
conda activate deeptan

pip install torch==2.7.0 torchvision==0.22.0 torchaudio==2.7.0 --index-url https://download.pytorch.org/whl/cu128
pip install torch_geometric
pip install pyg_lib torch_scatter torch_sparse torch_cluster torch_spline_conv -f https://data.pyg.org/whl/torch-2.7.0+cu128.html
pip install deeptan
```

## Usage

### 1. Build guide graphs with mi2graph
Before training DeepTAN, you first need to construct guide graphs from expression data.
This step is performed using mi2graph.

mi2graph installation and usage
Please follow the official repository:

https://github.com/WuChenh/mi2graph
This step typically produces graph files that DeepTAN can use downstream, especially:

training graph data in .npz format,
split-specific .parquet files for train/validation/test,
optional label files such as celltypes_onehot.parquet.

### 2. Convert graph data to litdata format
After building guide graphs, the next step is to optimize the dataset into litdata format for faster training and evaluation.

This stage uses:
run_04_litdata.py
run.04.litdata.optimize.sh

#### 2.1 What run_04_litdata.py does
run_04_litdata.py performs the following:
reads training graph data from a .npz file generated by mi2graph,
reads validation and test datasets from .parquet,
constructs a DeepTANDataModule,
exports train/val/test data into optimized litdata chunks,
saves metadata required for later model initialization.
The script saves the following metadata to the output directory:
litdata_others2save.json
litdata_others2save.pkl
These files contain:
dict_node_names
output_g_label_dim
If a label file is provided, it is also copied into the output directory.

#### 2.2 run_04_litdata.py arguments
Required arguments
--trn_npz
Path to training graph data in .npz format generated by mi2graph.
--val_parquet
Path to validation data in .parquet format.
--tst_parquet
Path to test data in .parquet format.
Optional arguments
--labels
Path to label data in .parquet format.
--output_dir
Output directory for optimized litdata files and metadata.
--bs
Batch size used when constructing the data module.
--thre_mi
Threshold for graph edge attributes.
--in_feat
Optional feature list file. If not specified, all features are used.
--in_obs
Optional training observation filter file.
If provided, only training cells whose obs_names are present in this file will be retained.
--onlytest
If enabled, only the test split is optimized.
--n_workers
Number of workers used during litdata optimization.

#### 2.3 How --in_obs works
If --in_obs is provided, the script:

reads all obs_names from the training .npz,
reads target obs_names from the provided parquet file,
computes their intersection,
keeps only the matching training indices when generating litdata.
This is especially useful when:

working with precomputed global graph files,
restricting training to a subset of cells,
running split-based or seed-based experiments.

A typical output directory looks like:

optimized_new/

### 3. Train or fine-tune DeepTAN
Once litdata has been prepared, you can launch DeepTAN training or hyperparameter tuning.

This stage uses:

run_05_fit_tune.py
run.local.fit.data_full_split42.sh

#### 3.1 What run_05_fit_tune.py does
run_05_fit_tune.py is the main fitting and tuning entrypoint. It:

loads data from a litdata directory,
builds the DeepTAN training configuration,
optionally loads an existing checkpoint,
supports task focus selection,
supports disabling guide-graph edge weights in GAT,
runs either:
direct training, or
Optuna-based hyperparameter tuning.
Internally it creates a DeepTANTune trainer and then calls:

trainer.optimize(...) if auto tuning is enabled,
trainer._train_on_args() otherwise.

#### 3.2 run_05_fit_tune.py arguments
Training and tuning mode
--auto_tune, --atune
Enable Optuna-based hyperparameter tuning.
Checkpoint and task mode
--em
Existing model checkpoint path to load.
--focus
Focus on a specific task. Supported values:
None
recon
label
recon_and_freeze
label_and_freeze
Guide graph option
--no_guide_gat, --nog
Disable using guide-graph edge weights in graph attention.
Data and logging
--litdata, --data
Path to litdata directory.
--log_dir, --logdir
Output directory for logs and training results.
Optimization and model settings
--bs
Batch size.
--lr
Learning rate.
--input_node_emb_dim, --indim
Input node embedding dimension.
--is_regression, --ir
Whether the task is regression.
--acc_grad_batch, --agb
Gradient accumulation steps.
--chunk_size, --ck
Chunk size balancing memory usage and speed.
Hardware settings
--accelerator, --ac
Hardware backend, e.g. cpu, gpu, mps, auto.
--devices, --dev
Devices to use. Can be a single device string or a list string like "[0]".
Optuna settings
--ntrials, --nt
Number of tuning trials.
--njobs, --nj
Number of parallel Optuna jobs.

###  4. End-to-end examples
Step 1: build guide graphs
Use mi2graph following:

https://github.com/WuChenh/mi2graph
Step 2: optimize data into litdata
python run_04_litdata.py \
  --trn_npz /path/to/split_47_0.npz \
  --val_parquet /path/to/split_47_1.parquet \
  --tst_parquet /path/to/split_47_2.parquet \
  --labels /path/to/celltypes_onehot.parquet \
  --output_dir /path/to/optimized/seed_47 \
  --in_obs /path/to/split_47_0.parquet \
  --n_workers 20
Step 3: tune or train DeepTAN
python run_05_fit_tune.py \
  --data /path/to/optimized/seed_47 \
  --logdir /path/to/logs/seed_47/multitask \
  --bs 256 \
  --nt 20 \
  --nj 1 \
  --agb 1 \
  --ck 32 \
  --dev "[0]" \
  --nog \
  --atune
Or simply run the shell wrappers:

bash run.04.litdata.optimize.sh
bash run.local.fit.data_full_split42.sh

###  5. Practical notes
#### 5.1 Keep metadata files
Do not remove the generated metadata files:

litdata_others2save.json
litdata_others2save.pkl
They contain structural information required by later stages.

#### 5.2 Label consistency
If you use classification or multitask settings, ensure that the label parquet file is aligned with the cell order and sample naming convention used in the dataset.

#### 5.3 Multi-split experiments
For reproducible experiments, it is recommended to store each seed or split in a separate directory, for example:

optimized_new/
└── ath_pretrain.full.nmicg8/
    ├── seed_42/
    ├── seed_43/
    ├── seed_44/
    ├── seed_45/
    └── seed_46/

#### 5.4 GPU memory tuning
For large datasets, the shell training script shows several useful environment variables for memory optimization, such as:

PYTORCH_CUDA_ALLOC_CONF
TOTAL_VRAM
TORCH_CUDNN_V8_API_ENABLED
TORCHINDUCTOR_FREEZING
You may need to adjust:

batch size,
chunk size,
gradient accumulation,
number of workers,
depending on your hardware.
